准备工作

1. github账户

  • 没有的话先去注册一个：https://github.com

  • 新建一个Repository，格式为: 用户名.github.io

2. 安装git

  • Ubuntu： sudo apt install git

  • Windows

    • 官网：https://git-scm.com/download/win
    • 淘宝镜像：https://npm.taobao.org/mirrors/git-for-windows/

安装后执行以下命令

git config --global user.name "username"  # github用户名，非昵称
git config --global user.email  "youremail@example.com"  # github注册邮箱

这两条命令会在主目录下创建 .gitconfig 账户配置文件

3. 创建SSH Key

  • 检查本机是否已存在ssh密钥：id_rsa和id_rsa.pub
    • Linux路径：~/.ssh，win10路径：%HOMEPATH%/.ssh
# 若不存在目录或这两个文件，就执行下面命令创建SSH Key
ssh-keygen -t rsa -C "youremail@example.com"
# 邮箱为github注册邮箱
# 一直回车就好了
# win10如果提示不是内部或外部命令需要到git安装路径/usr/bin 中打开cmd执行

  • github主页 → 头像 → Settings → SSH and GPG keys → New SSH key：

    • Title可以任意填

  • Key为id_rsa.pub文件的内容

    • Linux：终端执行cat ~/.ssh/id_rsa.pub
      • win10：用文本编辑器打开

4. 安装Nodejs

  • Linux：nvm (Node Version Manager)

    • nvm是 Nodejs 版本管理器，它让我们方便的管理Nodejs 版本。

    • 官方文档：https://github.com/nvm-sh/nvm

    • 用git安装nvm: 

      # 打开终端进入到主目录 cd ~/
git clone https://github.com/nvm-sh/nvm.git .nvm
cd ~/.nvm
git checkout v0.35.3 # 注意当前版本号
. nvm.sh

      在~/.bashrc、~/.profile 或 ~/.zshrc中添加

      export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"  # This loads nvm
[ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion"  # This loads nvm bash_completion

export NVM_NODEJS_ORG_MIRROR=http://npm.taobao.org/mirrors/node # 设置npm镜像源

    • 其他方法查看官方文档，或者 使用 nvm 管理不同版本的 node 与 npm

    • 常用命令

        nvm ls-remote  # 查看可安装的node版本
  nvm install [version] # 安装指定版本node
nvm uninstall [version] # 卸载指定版本的node
  nvm use [version] # 设置全局默认node版本
  nvm list # 查看本地已安装的node版本
  nvm current # 查看当前使用的node版本

  • Win10：nvs (Node Version Switcher)

    • 官方文档：https://github.com/jasongin/nvs

    • 在 nvs/releases 下载最新版本的 nvs.msi，然后双击安装即可。

    • 默认安装路径：%HOMEPATH%\AppData\Local\nvs

      nvs remote node https://npm.taobao.org/mirrors/node/ # 设置镜像源

    • 常用命令

      nvs ls-remote # 查看可安装的node版本
nvs ls # 查看本地已安装的node版本
nvs add [version] # 下载指定版本node
nvs rm <version> # 删除指定版本的node
nvs use [version] # 当前shell使用指定版本的node
nvs link [version] # 设置全局默认node版本
nvs unlink [version]

5. 安装Hexo

  • 官网：https://hexo.io/zh-cn/

  • 官方文档：https://hexo.io/zh-cn/docs/

    npm install hexo-cli -g

初始化

# 在当前目录下创建MyBlog文件夹并初始化
hexo init MyBlog

cd MyBlog
hexo g  #  generate
hexo s  #  server

也可以创建一个目录，然后在目录内打开终端输入hexo init

INFO  Start processing
INFO  Hexo is running at http://localhost:4000 . Press Ctrl+C to stop.

看到上面的输出信息，在浏览器打开http://localhost:4000就可以看到生成的第一篇Blog了

修改主题

hexo-theme-fluid

  • 一款 Material Design 风格的主题
  • 配置指南 | Fluid
  • 搭配 Fluid 如何优雅的写一篇文章

部署

先安装一个部署插件

npm install hexo-deployer-git --save

打开根目录下的 _config.yml 文件，找到 Deployment (最底部)

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: git@github.com:用户名/用户名.github.io.git
  branch: master

之后就可以执行部署命令了

helloworld的文章在source/_post，删除就不会显示了

hexo clean
hexo g
hexo d  #  deploy

部署完成之后就可以访问 GitHub Repository 同名的链接了，即：https://用户名.github.io
